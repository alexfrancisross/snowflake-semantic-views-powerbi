# Snowflake Semantic Views - Custom Connector Installation

This folder contains the custom connector for Power BI Desktop.

## File
- `SnowflakeSemanticViews.mez` - Power BI custom connector

## Installation Steps

### 1. Locate Your Custom Connectors Folder

**Windows:**
```
Documents\Power BI Desktop\Custom Connectors\
```

**Mac (via Parallels or Boot Camp):**
```
Documents/Power BI Desktop/Custom Connectors/
```

If the `Custom Connectors` folder doesn't exist, create it.

### 2. Copy the Connector

Copy `SnowflakeSemanticViews.mez` to the Custom Connectors folder.

### 3. Enable Custom Connectors in Power BI Desktop

1. Open Power BI Desktop
2. Go to **File > Options and settings > Options**
3. Select **Security** from the left menu
4. Under **Data Extensions**, select:
   - "(Not Recommended) Allow any extension to load without validation or warning"
5. Click **OK**

### 4. Restart Power BI Desktop

Close and reopen Power BI Desktop for the connector to be loaded.

### 5. Verify Installation

1. Click **Get Data**
2. Search for "Snowflake Semantic Views"
3. The connector should appear in the list

## Using the Connector

1. Click **Get Data** > **Snowflake Semantic Views**
2. Enter your Snowflake server URL (e.g., `myaccount.snowflakecomputing.com`)
3. Enter your warehouse name
4. Click **Connect**
5. Enter your Snowflake credentials
6. Navigate to your semantic view: Database > Schema > View
7. Select **DirectQuery** mode for real-time data
8. Click **Load**

## Troubleshooting

### Connector Not Appearing
- Verify the .mez file is in the correct folder
- Check that custom connectors are enabled in Options > Security
- Restart Power BI Desktop completely

### Authentication Errors
- Verify your Snowflake credentials
- Check that your user has access to the specified warehouse
- Ensure your account isn't requiring MFA (use SSO if available)

### Data Access Errors
- Verify you have access to the database/schema/semantic view
- Check that the semantic view exists and is accessible
- Ensure your Snowflake warehouse is running (not suspended)
